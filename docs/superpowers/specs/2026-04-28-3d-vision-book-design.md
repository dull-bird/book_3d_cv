# 3D 视觉开源书：设计文档

## 项目定位

- **书名（暂定）**：《3D 视觉：从原理到实践》
- **语言**：中文
- **核心目标**：总结近 10 年 3D 视觉 AI 领域的重要学术成果，用通俗语言讲解原理，覆盖从入门到部署的全流程，帮助资源有限的每个人理解和上手
- **读者定位**：不限背景，覆盖全水平（入门 → 算法改进 → 生产部署）
- **写作理念**：用写书来驱动学习，先产出 MVP（4 模块），后续持续扩充
- **协作模式**：研究先行 → 作者提供一手文献 → AI 基于文献生成初稿 → 作者负责技术准确性审查和风格统一。AI 不得在缺乏一手文献支撑的情况下凭空生成技术内容

## 研究准则：一手文献先行

**这是本书的根基约束，高于一切写作流程。**

### 原则

- **没有一手文献，不落笔**——每个模块的研究阶段必须先于写作阶段
- **追溯原文**——公式推导、算法细节必须来自原版论文或原版教材，不依赖二手解读
- **英文原版优先**——核心教材和论文以英文原版为准
- **代码验证**——关键算法的理解必须对照官方开源实现

### 每个模块的研究清单

在开始写任何一个模块之前，必须先收集并研读：

1. **奠基性教材/专著**（如基础篇依赖 *Multiple View Geometry in Computer Vision* by Hartley & Zisserman）
2. **里程碑论文**（该方向 3-5 篇最具影响力的，原文全文）
3. **SOTA 论文**（当前最好的方法，含官方代码仓库）
4. **基准与数据集**（该领域的标准评测基准及其元数据）

### 研究输出

每个模块研究阶段结束后，产出：
- 论文阅读笔记（摘要 + 关键公式 + 核心 insight）
- 该模块的"文献地图"——一张 mindmap 或 flowchart 展示论文之间的演进关系
- 代码跑通记录（关键模型确实在自己的环境里跑过）

这些研究笔记存放在 `20_Projects/book_3d_cv/research/` 下，作为写作的原材料。

## 书籍架构

### 两大部分

**PART 0 — 基础篇（不变的脚手架）**

所有主题模块的前置知识，一次性写好，后续模块只引用不重复：

- 相机模型（针孔相机、内参外参、畸变）
- 投影几何（3D→2D，齐次坐标）
- 多视图几何入门（对极几何、三角化、本质矩阵/基础矩阵）
- 深度表示（视差图、深度图、点云的优缺点）
- 坐标系转换（世界/相机/像素坐标统一图）
- 优化基础（最小二乘、RANSAC、Bundle Adjustment 直觉）
- Mini Case：用手机拍两张照片，手动算距离

穿插思考题和 Mini Case 降低枯燥感。

**PART 1-N — 主题模块（持续生长）**

每个模块独立成篇，通过 Wiki 链接引用基础篇和交叉引用其他模块。按版本渐进发布：

- **v0.1 (MVP)**：模块 A「单目深度估计」+ 模块 B「双目立体匹配」+ 模块 C「3D Gaussian Splatting」
- **后续扩展**：深度补全、NeRF、VGGT、SLAM、SfM、点云处理、MVS 等

### 设计原则

- **模块自包含**：每篇独立可读，不要求线性阅读
- **版本化发布**：v0.1 = 基础篇 + 3 个 MVP 模块，v0.2 加一篇，以此类推
- **AI 起草 + 人工把关**：AI 生成初稿，作者负责技术审查
- **交叉引用**：模块间以 Wiki 链接互相支撑

## 模块模板：由浅入深三阶 + 苏格拉底时刻

每个主题模块严格按此结构展开。核心原则：**钩子先行，推导紧跟**——永远先让人知道"为什么要学"，再展开严密的推导。

### 第一阶：直观理解（给所有人）

- 一个具体的痛点场景开头
- 用类比建立直觉，不做数学推导
- 一张 mindmap 展示技术全景
- 一张 timeline flowchart 展示 10 年关键突破
- Mini Case：用现成模型跑出结果

### 第二阶：原理解析（给做算法的人）

采用"第一性原理引擎"：
1. 剥离表象 → 这个问题的物理/几何本质是什么？
2. 长链推演 → 从基本假设逐步推导到 SOTA 方法
3. Mermaid 图表辅助（flowchart 展示算法逻辑，xychart 对比方法性能）
4. 关键公式配"人话翻译"
5. Code Lens：核心代码段逐行解释

### 第三阶：部署实战（给要落地的人）

- 模型选型 quadrantChart（精度 × 速度四象限）
- 战争故事：真实部署中踩过的坑
- 数据问题：训练集偏差、域迁移、评估指标陷阱
- 端到端案例：完整 pipeline

### 章末：苏格拉底时刻

每章末尾 1-2 个直击本质的反问，替代传统的"本章小结"。激发读者的逆向和深度思考。

### 章末固定元素

- 关键论文清单（按时间线排列，每篇一句话贡献）
- 实操练习（跑模型 + 调参 + 观察失败 case）
- 延伸阅读（交叉引用 + 进阶资料）

## MVP 模块内容大纲

### 基础篇：3D 视觉的脚手架

1. 相机模型：针孔相机、内参外参、畸变——一张图讲清楚光路
2. 投影几何：3D→2D 的数学，齐次坐标为什么好用
3. 多视图几何入门：对极几何、三角化、本质矩阵和基础矩阵
4. 深度表示：视差图、深度图、点云，各自的优缺点
5. 坐标系：世界、相机、像素坐标之间的转换——一张总图
6. 优化基础：最小二乘、RANSAC、bundle adjustment 直觉
7. Mini Case：用手机拍两张照片，手动算出桌子的距离

### 模块 A：单目深度估计

1. 问题定义：一张 RGB 图 → 每个像素的深度值，本质是 ill-posed
2. 经典方法：Shape from Shading, 纹理梯度——为什么不够用
3. 深度学习时代：Eigen et al. (2014) → Monodepth → MiDaS → Depth Anything → Depth Pro
4. 关键洞察：从监督到自监督到 foundation model 的范式转移
5. Loss 的演变：尺度不变损失、多尺度梯度、法线损失
6. 部署：单目模型在机器人/AR 中的实际使用和局限性

### 模块 B：双目立体匹配

1. 问题定义：两张矫正后的图像 → 逐像素视差，三角化得深度
2. 经典管线：匹配代价 → 代价聚合 → 视差计算 → 视差精化
3. 深度学习革命：DispNet → PSMNet → RAFT-Stereo → FoundationStereo
4. 核心挑战：遮挡、无纹理区域、重复纹理、反射表面
5. Gaussian Curvature 视角：低曲率作为几何先验
6. 部署：实时双目系统在机器人/自动驾驶中的权衡

### 模块 C：3D Gaussian Splatting

1. 问题定义：从多张照片重建可渲染的 3D 场景
2. 前身 NeRF：隐式表示 + 体渲染，为什么慢
3. 3DGS 的突破：显式点云 + 可微光栅化 = 实时渲染 + SOTA 质量
4. 第一性原理：为什么椭球体（Gaussian）？——各向异性 + 可微性好
5. 关键算法：初始化、克隆/分裂、致密化、SH 颜色
6. 前沿变体：4DGS、Scaffold-GS、2DGS
7. 部署：训练自己的 3DGS 场景，在手机/Web 上渲染

## 模块间交叉引用

- 基础篇 ← 被所有模块引用（前置依赖）
- 单目(A) ↔ 双目(B)：单目自监督依赖双目做监督信号；FoundationStereo 融合了单目特征
- 双目(B) → 3DGS(C)：多视图立体匹配是 3DGS 初始化的上游
- 单目(A) → 3DGS(C)：单目深度 priors 正被用于改进 3DGS 几何一致性
