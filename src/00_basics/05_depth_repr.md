# 第 5 章 深度表示：三维数据到底长什么样

> [!NOTE]
> **预计阅读时间**：40 分钟 · **前置知识**：第 1、2、4 章
>
> 3D 视觉的输出不只有一种形态。它可以是一张深度图、一张视差图、一团点云、一个体素网格、一个 mesh，甚至是神经场。本章目标是建立这些表示之间的转换关系和适用边界。

---

## 5.1 本章目标

第 4 章讲了如何从两张图恢复三维点。但恢复出来的数据到底怎么存？

常见表示包括：

| 表示 | 长什么样 | 最适合 |
|------|---------|--------|
| 深度图 | 每个像素一个深度值 | RGB-D、单目深度、稠密重建 |
| 视差图 | 每个像素一个左右图横向位移 | 双目立体匹配 |
| 点云 | 一堆 $(X,Y,Z)$ 点 | 几何处理、配准、测量 |
| Mesh | 顶点 + 三角面 | 渲染、碰撞、CAD |
| 体素/TSDF | 规则 3D 网格 | 多帧融合、重建表面 |
| NeRF/3DGS | 连续/显式可渲染场 | 新视角合成、场景重建 |

核心问题不是“哪种最好”，而是：**你的任务需要哪种表示，以及它会丢掉什么信息。**

---

## 5.2 深度图：每个像素一把尺子

深度图（Depth Map）和 RGB 图一样是二维数组，只是每个像素存的不是颜色，而是距离。

通常定义：

$$D(u,v)=Z_c$$

也就是该像素对应三维点在相机坐标系下的 $Z$ 值。

> [!CAUTION]
> 深度不一定等于欧氏距离。很多数据集和 RGB-D 相机给的是 $Z_c$，即沿光轴方向的深度；而不是点到相机中心的直线距离 $\sqrt{X_c^2+Y_c^2+Z_c^2}$。

深度图优点：

- 和图像像素一一对齐，适合 CNN/Transformer。
- 能直接和 RGB 拼成 RGB-D。
- 存储紧凑，适合实时系统。

局限：

- 只能描述当前视角可见表面，是 2.5D。
- 看不到物体背面和遮挡后的结构。
- 边缘处容易混合前景/背景深度。

实际工程里，深度图还要先问清楚三个细节：

| 问题 | 为什么重要 |
|------|-----------|
| 单位是什么 | 有的深度图用米，有的用毫米，有的 16-bit PNG 存的是毫米整数 |
| 无效值怎么编码 | 常见是 0、NaN、最大值或单独的 mask |
| 是否和 RGB 对齐 | 深度相机和 RGB 相机通常不是同一个光心，需要对齐后才能逐像素融合 |

> [!TIP]
> 很多 RGB-D bug 看起来像算法问题，其实是单位、无效值或对齐方式错了。读入一张深度图后，先打印最小值、最大值、无效比例，再可视化一遍。

## 5.3 从深度图到点云：反投影

已知内参 $K$ 和深度 $Z=D(u,v)$，可以把像素反投影到相机坐标系：

$$x = (u-c_x)/f_x,\qquad y=(v-c_y)/f_y$$

$$X_c=Zx,\qquad Y_c=Zy,\qquad Z_c=Z$$

代码：

```python
import numpy as np


def depth_to_points(depth, K):
    """
    Convert a depth map to camera-frame point cloud.
    depth: HxW, depth in meters, invalid pixels <= 0.
    K: 3x3 intrinsic matrix.
    """
    h, w = depth.shape
    fx, fy = K[0, 0], K[1, 1]
    cx, cy = K[0, 2], K[1, 2]

    u, v = np.meshgrid(np.arange(w), np.arange(h))
    z = depth
    valid = z > 0

    x = (u[valid] - cx) / fx * z[valid]
    y = (v[valid] - cy) / fy * z[valid]
    points = np.column_stack([x, y, z[valid]])
    return points
```

如果还知道外参，可转到世界坐标：

$$X_w = R^T(X_c-t)$$

> [!TIP]
> 深度图到点云的过程，就是第 2 章“像素反投影射线”补上深度 $\lambda=Z$。

## 5.4 视差图：双目里的深度表示

双目矫正后，对应点在同一行：

$$d = u_L-u_R$$

$d$ 叫**视差**（disparity）。深度和视差关系：

$$Z=\frac{fB}{d}$$

其中：

- $f$：像素焦距
- $B$：双目基线
- $d$：视差，单位 pixel

> [!TIP]
> 视差越大，物体越近；视差越小，物体越远。远处点的视差接近 0，所以远距离深度误差会急剧放大。

误差传播近似：

$$\Delta Z \approx \frac{Z^2}{fB}\Delta d$$

这说明：

- 距离越远，深度越不准。
- 焦距越大，深度越准。
- 基线越大，深度越准。
- 匹配精度越高，深度越准。

## 5.5 点云：无序的三维点集合

点云（Point Cloud）是很多 3D 传感器和重建算法的通用输出：

$$\{(X_i,Y_i,Z_i)\}_{i=1}^N$$

点还可以带属性：

| 属性 | 用途 |
|------|------|
| RGB | 彩色点云、可视化 |
| normal | 表面重建、配准 |
| intensity | LiDAR 反射强度 |
| confidence | 过滤低质量点 |
| timestamp | 多传感器同步 |

点云优点：

- 直接在 3D 空间中表达几何。
- 容易做距离测量、裁剪、配准。
- 适合 LiDAR、结构光、RGB-D 输出。

局限：

- 无序，不适合普通 2D 卷积。
- 没有表面连接关系。
- 密度不均匀，近处密、远处稀。

点云的“无序”不是说它乱，而是说数组顺序本身没有几何意义。第 100 个点和第 101 个点在内存里挨着，不代表它们在空间中相邻。算法如果需要邻域关系，通常要先建 KD-tree、体素网格或 range image。

### 点云常见处理

| 操作 | 目的 |
|------|------|
| 体素下采样 | 降低点数，均匀化密度 |
| 法线估计 | 获取局部表面方向 |
| 离群点过滤 | 去掉孤立噪声 |
| ICP 配准 | 对齐两帧点云 |
| 表面重建 | 从点变成 mesh |

Open3D 示例：

```python
import open3d as o3d

pcd = o3d.io.read_point_cloud("scene.ply")
pcd = pcd.voxel_down_sample(voxel_size=0.02)
pcd.estimate_normals()
o3d.visualization.draw_geometries([pcd])
```

## 5.6 Mesh：点之间连成面

Mesh 用顶点和面片表示表面：

```text
vertices: (N, 3)
faces:    (M, 3)  # 每个三角形引用三个顶点
```

相比点云，mesh 多了拓扑连接关系，所以更适合：

- 渲染
- 碰撞检测
- 3D 打印
- CAD/逆向工程

但 mesh 也更难生成。点云噪声、孔洞、稀疏区域都会导致重建面片出错。

常见重建方法：

| 方法 | 特点 |
|------|------|
| Poisson Reconstruction | 平滑、能补洞，但可能过度补全 |
| Ball Pivoting | 保留局部细节，依赖点密度 |
| Delaunay/Alpha Shape | 几何直观，参数敏感 |

## 5.7 体素、Occupancy 和 TSDF

点云和 mesh 都是不规则结构。体素（Voxel）把空间切成规则三维网格：

```text
voxel[x, y, z]
```

每个格子可以存不同含义：

| 表示 | 每个体素存什么 | 用途 |
|------|----------------|------|
| Occupancy | 是否被占据 | 机器人避障、地图 |
| TSDF | 到最近表面的带符号距离 | 多帧深度融合、表面重建 |
| Semantic Voxel | 类别概率 | 语义地图 |

TSDF（Truncated Signed Distance Function）尤其重要。它存的是体素中心到表面的距离：

- 表面外侧为正。
- 表面内侧为负。
- 接近 0 的位置就是表面。
- 超过截断距离就不再精确存。

为什么 TSDF 适合融合多帧深度？因为单帧深度只告诉你“这条相机射线上某个位置是表面”，而多帧从不同角度观察同一块区域时，可以把这些表面证据累积到同一个体素网格里。噪声会被平均掉，孔洞也可能被其他视角补上。

> [!TIP]
> KinectFusion 这类系统的核心就是把每一帧深度图融合进 TSDF 体素网格，最后从零等值面提取 mesh。

局限是内存。规则体素分辨率翻倍，体素数变 8 倍，所以大场景通常需要稀疏体素、八叉树或哈希体素。

## 5.8 神经场和 3DGS：现代可渲染表示

现代 3D 重建不只输出传统几何，也会输出可渲染场景表示。

| 表示 | 核心思想 | 优点 | 局限 |
|------|---------|------|------|
| NeRF | 用神经网络表示连续密度和颜色场 | 视角合成质量高 | 训练/渲染成本较高 |
| 3D Gaussian Splatting | 用大量带颜色和透明度的 3D 高斯表示场景 | 实时渲染强 | 几何表面不如 mesh 明确 |

它们和前面的几何表示并不冲突。相机内外参、投影、深度、点云仍然是训练和评估这些方法的基础。

---

## 5.9 表示之间怎么选

| 任务 | 推荐表示 |
|------|---------|
| 单目深度估计 | 深度图 |
| 双目立体匹配 | 视差图，必要时转深度图 |
| RGB-D 感知 | 对齐 RGB-D + 点云 |
| 机器人导航 | Occupancy / ESDF / 稀疏点云 |
| 物体尺寸测量 | 点云或 mesh |
| 新视角合成 | NeRF / 3DGS |
| 3D 打印 | Watertight mesh |
| 多帧融合 | TSDF / surfel / voxel hash |

## 5.10 本章小结、问题与练习

最重要的三句话：

1. 深度图是 2.5D，点云是无序 3D，mesh 是带拓扑的表面。
2. 双目视差和深度满足 $Z=fB/d$，远处深度误差会平方级放大。
3. 没有一种表示适合所有任务，表示选择会决定后续算法形态。

### 苏格拉底时刻

1. 为什么深度图看不到物体背面？
2. 为什么同样的视差误差，在远处会造成更大的深度误差？
3. 点云和 mesh 的区别是什么？为什么点云不能直接用于 3D 打印？
4. TSDF 为什么适合多帧融合？

### 实操练习

**练习 1：深度图转点云**

找一张 16-bit 深度图，按本章代码转成点云，观察无效深度和边缘噪声。

**练习 2：视差转深度**

设 $f=800$ pixel，$B=0.1$ m，计算视差 $d=80,40,20,10$ 时的深度。观察深度如何随视差变化。

**练习 3：点云下采样**

用 Open3D 对同一份点云做不同 voxel size 的下采样，比较点数、细节和处理速度。
