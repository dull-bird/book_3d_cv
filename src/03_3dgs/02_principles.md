# C.2 原理解析

> **目标**：理解 3DGS 的三个核心组件——3D 高斯表示、自适应密度控制、tile-based 快速可微光栅化器。知道 Mip-Splatting、Scaffold-GS、2DGS 各解决了什么问题。

## 3D 高斯的数学表示

3DGS 的场景由 $N$ 个 3D 高斯椭球体组成。每个高斯 $i$ 由以下参数定义：

| 参数 | 符号 | 维度 | 含义 |
|------|------|------|------|
| 位置（均值） | $\mu_i$ | $3$ | 高斯中心在 3D 空间的位置 |
| 协方差矩阵 | $\Sigma_i$ | $3 \times 3$ | 椭球的形状、大小和朝向 |
| 不透明度 | $\alpha_i$ | $1$ | 0（完全透明）到 1（完全不透明） |
| 球谐系数 | $SH_i$ | $16 \times 3$（1 阶）或 $4 \times 3$（3 阶） | 随视角变化的颜色 |

协方差矩阵 $\Sigma$ 被分解为**缩放矩阵 $S$ + 旋转矩阵 $R$**，由四元数 $q$ 参数化——这是确保 $\Sigma$ 始终是正定矩阵的最便宜方式：

$$\Sigma = RSS^T R^T$$

其中 $S = \text{diag}(s_x, s_y, s_z)$ 是三个轴的缩放因子，$R$ 是由四元数 $q$ 构造的旋转矩阵。这种分解保证 $\Sigma$ 在任何参数值下都是合法的协方差。

## Tile-Based 快速可微光栅化器

这是 3DGS 论文实现实时渲染的核心工程贡献。不同于 NeRF 的逐射线体积渲染，3DGS 的渲染器基于 GPU 光栅化。

### 第一阶段：剔除与预排序

1. **屏幕分块**：将图像划分为 $16 \times 16$ 的 tile 网格。
2. **视锥剔除**：剔除投影后完全不在屏幕内的 3D 高斯（或置信区间 < 99% 的）。
3. **实例化**：每个高斯按它覆盖的 tile 数量，复制对应的实例（一个高斯覆盖 3 个 tile → 复制 3 份）。
4. **分配 Key**：每个实例被分配一个 Key，由 **视图空间深度 + tile ID** 组成。
5. **GPU Radix Sort**：按 Key 一次性全局排序——排序后的序列中，每个 tile 内的高斯从前到后排列。

**关键设计**：全图一遍排序，不逐像素排序。利用了 GPU 的并行排序能力（NVIDIA CUB 库），开销远低于传统逐像素排序。

### 第二阶段：Tile 级并行渲染

每个 tile 分配一个 CUDA 线程块：
- 协作加载高斯的投影参数（2D 均值 + 2D 协方差 + α + 颜色）到**共享内存**。
- 按排序列表从前到后累积颜色和不透明度：$C = \sum_{i} c_i \alpha_i \prod_{j < i} (1 - \alpha_j)$。
- 像素的 α 接近饱和（$\approx 1$）时，该像素的线程提前停止。
- 整个 tile 的所有像素都饱和后，tile 整体终止。

**为什么快**：不限制参与混合的高斯数量（能适应任意深度复杂度的场景）。每个像素只需常数级额外显存。利用 tile 内像素的局部性——相邻像素看到的高斯高度重叠，共享内存命中率高。

### 第三阶段：可微反向传播

前向渲染时只存一个值：每个像素的**最终累积 α**。反向传播时，利用这个值重建中间不透明度——从后向前遍历，每步的中间 α 可以通过最终的 α 除以当前高斯的透射率恢复。这避免了存储每个像素的完整混合列表（否则显存会爆炸）。

所有梯度（$\mu, \Sigma, \alpha, SH$）都被显式推导并实现为 CUDA kernel，不依赖 PyTorch 的自动微分——这是 3DGS 训练速度的另一个关键因素。

### 渲染公式

3D 高斯投影到 2D 后成为一个 2D 高斯光斑（splat），其 2D 协方差为：

$$\Sigma' = J W \Sigma W^T J^T$$

其中 $W$ 是世界到相机的视图变换，$J$ 是投影变换的雅可比矩阵（affine approximation of perspective projection）。每个像素的颜色由覆盖该像素的 $N$ 个高斯按深度排序后 α 混合：

$$C = \sum_{i=1}^{N} c_i \alpha_i \prod_{j=1}^{i-1} (1 - \alpha_j)$$

其中 $c_i$ 是从球谐系数 $SH_i$ 在特定视角方向 $d$ 上计算的颜色，$\alpha_i$ 是高斯在像素位置的 2D 不透明度贡献。

## 自适应密度控制

3DGS 从 SfM 稀疏点云（通常几千个点）开始，通过优化过程中动态增删高斯来达到几百万个高质量椭球体。

**增长策略（欠重建）**：在视空间位置梯度大的区域——说明场景在这里有比当前高斯更精细的结构——有两种操作：
- **克隆（Clone）**：对于小的高斯，直接复制一份。适合需要更多高斯来覆盖的细节区域。
- **分裂（Split）**：对于大的高斯，沿最大缩放轴一分为二（两个新高斯的大小按 1.6 倍缩小）。适合覆盖过大、模糊的单个高斯。

**修剪策略（过重建）**：定期移除两类高斯：
- α 接近 0 的——基本透明，对渲染无贡献。
- 世界空间过大的——可能覆盖了不该覆盖的区域。

此外，每 3000 次迭代将全部高斯的 α 重置为接近 0，防止优化过程中被少数早期"错误"的高斯占据。密度控制在前 15000 次迭代中最活跃，之后趋于稳定。

## 损失函数

3DGS 的优化目标非常简单——只用 L1 和 SSIM：

$$\mathcal{L} = (1 - \lambda)\mathcal{L}_1 + \lambda\mathcal{L}_{\text{D-SSIM}}$$

默认 $\lambda = 0.2$。没有 3D 几何监督、没有法向量约束——纯粹的渲染 loss。这也是 3DGS 几何质量差的根本原因——模型只被激励去"画好看"，不关心"形状对不对"。

## 重要改进

### Mip-Splatting（CVPR 2024 Oral, Best Student Paper）

原始 3DGS 在缩放或改变焦距时会产生**锯齿**——当 3D 高斯投影到屏幕上已小于一个像素时，高频噪声出现。Mip-Splatting 加了两个滤波：

1. **3D 平滑滤波**：根据训练视角的最大采样频率（最远视角下每个像素覆盖的物理尺寸），限制 3D 高斯的尺寸下限——确保没有高斯小到失去意义。
2. **2D Mip 滤波**：用模拟 2D box filter 替代原始的 dilation 操作，在投影后对 2D 高斯做抗锯齿。

### Scaffold-GS（CVPR 2024 Highlight）

原始 3DGS 的每个高斯是独立优化、自由漂移的——容易出现"散架"（Gaussians 漂到不该去的地方）。Scaffold-GS 用**锚点**从稀疏体素网格组织高斯：

- 每个锚点 spawn $k$ 个 neural Gaussian，属性由轻量 MLP 预测（输入：锚点特征 + 视角方向 + 距离）。
- 锚点本身按梯度显著性和累积 α 做生长/修剪。
- 结果：可比质量、~100 FPS、更小存储、对反射/透明更鲁棒。

### 2D Gaussian Splatting（2024）

原始 3DGS 用 3D 体积椭球——没有"表面"概念，无法提取高质量网格。2DGS 把每个高斯替换为**2D 平面高斯盘**（flat disk），投影时用射线-平面求交（而非近似投影），并加强 normal consistency 和 depth distortion 正则化。在 DTU 和 Tanks & Temples 上达到 SOTA 的表面重建质量。
