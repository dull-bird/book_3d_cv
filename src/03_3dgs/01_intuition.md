# C.1 直观理解

> **要回答的问题**：3D Gaussian Splatting 到底在做什么？为什么它能替代 NeRF？为什么椭球体比神经网络更适合做渲染？"Splatting"这个词是什么意思？

## 一个场景

你用手机绕着桌上的盆栽拍了 50 张不同角度的照片。现在你想做到两件事：

1. **在任意新角度看到这盆盆栽**（比如从正上方往下看，但你并没有拍这个角度）。
2. **以 60 FPS 实时旋转视角**——像打游戏一样流畅。

传统做法：先用 COLMAP（SfM）恢复相机位姿和稀疏 3D 点云，然后用这些信息做多视图立体匹配（MVS）生成稠密点云，再用泊松重建生成网格，贴上纹理 → 得到一个带纹理的 3D 模型。这个流程极其漫长且 fragile——每一步都可能失败，而且最终的纹理质量往往很差。

2020 年的 NeRF 给了一个完全不同的方案：用一个神经网络记住整个场景的"辐射场"——输入是 3D 坐标 + 视角方向，输出是该点的颜色和密度。**但 NeRF 太慢了**——渲染一个像素需要沿射线采样几十个点，每个点都要调用神经网络。一张 1080p 图像需要几千万次网络前向——几秒甚至几十秒一帧。

## 核心直觉：从"体积采样"到"直接投影"

3DGS 的核心直觉是：**把场景表示为一堆可见的椭球体，直接投影到屏幕上，跳过体积采样。**

```
NeRF 的做法：                        3DGS 的做法：
像素 ← 沿射线采样100个点 ← 神经网络     像素 ← 投影一堆椭球 ← 排序 ← α混合
     每个点跑一次网络                         每个椭球投影一次
     慢（几千亿次网络调用）                   快（GPU光栅化）
     隐式（你看不到"场景长什么样"）           显式（椭球体就是场景）
```

**什么是 "Splatting"？** Splatting 是计算机图形学中的一个经典概念：不是从每个像素出发去"找"它对应的 3D 内容（ray tracing），而是反过来——把 3D 的内容直接"拍"到屏幕上的对应位置。一个 3D 高斯椭球体"拍"到屏幕上就是一个 2D 椭圆光斑（splat）。把所有椭球体的光斑从前到后叠起来（α 混合），就得到最终的图像。

> [!TIP]
> **NeRF 是"射线法"（从像素出发去找颜色），Splatting 是"投影法"（从 3D 物体出发投到屏幕上）。** Splatting 天然适合 GPU 并行——每个椭球体独立投影，不需要等别人的结果。这就是 3DGS 能跑到 100+ FPS 的根本原因。

## 3DGS 的三个关键设计选择

### 1. 为什么是"高斯"椭球体？

一个 3D 高斯（Gaussian）由位置 $\mu$（中心在哪）和协方差矩阵 $\Sigma$（椭球的形状、大小、朝向）定义。用高斯有几个好处：

- **自然适合 α 混合**：3D 高斯投影到 2D 后还是一个高斯（近似），两个重叠的高斯做 α 混合在数学上很干净。
- **各向异性**：协方差矩阵让每个高斯可以有自己的朝向和拉伸程度——长条形的高斯适合表示墙壁、地板等平面；球形的高斯适合表示孤立的物体。
- **连续可微**：高斯函数处处平滑、可微——梯度可以从渲染 loss 一直回传到高斯的参数。这对优化至关重要。

### 2. 为什么不直接用点云？

从 COLMAP 出来的稀疏点云（通常几千个点）直接渲染效果很差——点之间的空隙会让图像全是洞。3DGS 的做法是：以稀疏点云为"种子"，在每个点上放一个初始高斯，然后在优化过程中**自适应地增删高斯**——覆盖不够的地方复制新的高斯，覆盖太密了就把大高斯拆成两个小的。经过几万次迭代，从几千个高斯扩展到几百万个高斯，覆盖整个场景。

### 3. 为什么不需要神经网络？

NeRF 用一个 MLP 保存整个场景的颜色和密度。3DGS 把颜色存在每个高斯自己的参数里（球谐系数 SH，随视角变化）和不透明度（α）。渲染时不需要任何网络前向——把高斯投影到屏幕、排序、做 α 混合。整个过程是**纯计算**（矩阵运算 + 光栅化），没有神经网络的黑盒。这带来了两个好处：

- **速度**：GPU 光栅化极快（这是显卡的看家本领）。
- **可解释性**：场景就是一堆椭球体——你可以直接旋转、缩放、删除任何一个高斯。这对 3D 编辑至关重要。

## 3DGS 的局限

- **训练需要 COLMAP 位姿**：不像 DUSt3R 可以无标定重建，3DGS 需要精确的相机位姿。如果 SfM 失败了（比如场景全是白墙，特征匹配不上），3DGS 就无法训练。
- **显存需求大**：几百万个高斯 = 几百 MB 到几 GB 的显存。大场景需要更高效的高斯表示（Scaffold-GS 等改进方向）。
- **几何质量差**：原始 3DGS 的 3D 高斯没有"表面"概念——它们是体积元素，不是表面元素。提取网格需要额外的处理（如 2D GS 或 SuGaR）。
- **对新视角的泛化能力有限**：3DGS 是在训练视角上优化的——如果你要看的视角离训练视角太远（比如跑到盆栽背面但没拍过背面），质量会显著下降。
